# 三层仿真环境移植规划

## 目标

参考 `termitech_cloth_sim/sim/env/` 的三层架构（`BaseEnv → ClothEnvBase → ClothEnvARX`），在 `newton_physic_sim/sim/` 中构建一套独立的仿真环境框架。

**核心差异：**
- 不使用 Hydra/OmegaConf，用纯 Python dataclass 管理配置
- 不使用 `eval(name)` 动态实例化，改用字符串到类的注册表
- 先只做 `num_envs=1`，多环境后续再加
- 先不做 RL（task/vec_env），只做仿真 + 可视化
- 目标 Python 版本 3.12，使用 uv 管理依赖（与本项目一致）

---

## 目录结构（规划）

```
newton_physic_sim/sim/
  tutorial/               # 已有教程（不变）
  experiment/             # 已有实验脚本（不变）
  env/                    # 新增：三层环境类
    __init__.py
    base_env.py           # L1: 通用 Newton 环境外壳
    cloth_env_base.py     # L2: 机械臂+衣物仿真基类
    cloth_env_arx.py      # L3: ARX-X5 机器人定制
  controller/             # 新增：控制器
    __init__.py
    base_controller.py    # 控制器基类
    joint_pd_controller.py
    ik_controller.py      # 可选，先用 joint_pd
  renderer/               # 新增：渲染器
    __init__.py
    base_renderer.py
    default_renderer.py
  utils/                  # 新增：工具函数
    __init__.py
```

**与已有代码的关系：** `tutorial/` 和 `experiment/` 中的现有脚本不受影响，新框架在 `env/`、`controller/`、`renderer/`、`utils/` 中独立构建。

## 三层架构设计

### 第一层：BaseEnv — 通用 Newton 环境外壳

**文件：** `sim/env/base_env.py`

**职责：** 封装 Newton 引擎基础设施，不涉及任何机器人或衣物的具体逻辑。

**需要包含的成员：**

| 成员 | 类型 | 来源 |
|------|------|------|
| `viewer` | `ViewerGL` | `cfg.headless` 决定是否 headless |
| `model` | `Model` | 子类构建，初始 `None` |
| `state_0 / state_1` | `State` | 子类构建，初始 `None` |
| `sim_time` | `float` | 0.0 |
| `num_envs` | `int` | 从 cfg 读取 |
| `graph` | `CUDA Graph` 或 `None` | 由 `capture()` 生成 |
| `controller` | `BaseController` | 由 `setup_controller()` 创建 |
| `renderer` | `BaseRenderer` | 由 `setup_renderer()` 创建 |

**需要移植的方法：**

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| `__init__(cfg)` | 低 | 保存 cfg，创建 viewer |
| `initialize_resources(no_renderer)` | 低 | 调用 setup_renderer + setup_controller |
| `setup_controller()` | 低 | 用注册表创建控制器（不传 cfg 给基类接口） |
| `setup_renderer()` | 低 | 用注册表创建渲染器 |
| `capture()` | 中 | CUDA Graph 捕获，需 `cfg.use_graph` |
| `step(action)` | 低 | controller.compute → simulate（graph 或直调） |
| `render()` | 低 | 委托给 renderer.render() |
| `simulate()` | — | **抽象方法**，由子类实现 |
| `get_live_meshes()` | 高 | 直接移植，聚合变形体+刚体 mesh 数据 |
| `_get_shape_local_mesh()` | 高 | 直接移植，按几何类型生成可视化网格（带缓存） |
| `reset()` | 低 | 空壳，由子类覆写 |

**不移植的内容：**
- `task` 相关逻辑（RL 不需要）
- `OmegaConf` 类型注解
- `np.atan2 = np.arctan2` / `np.pow = np.power` monkey patch（Python 3.12 不需要）
- `import OpenGL.GL`、`import ctypes`、`import time`（未使用的 import）
- `sys.path.append` hack

### 第二层：ClothEnvBase — 机械臂+衣物仿真基类

**文件：** `sim/env/cloth_env_base.py`

**职责：** 构建包含机械臂、衣物、刚体的完整仿真场景，实现双求解器步进循环。这是工作量最大的一层。

**需要包含的成员（新增）：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `num_robot` | `int` | 机械臂数量 |
| `endeffector_id` | `int` | EE body 索引 |
| `num_arm_joints` | `int` | 臂关节数（6） |
| `num_gripper_joints` | `int` | 夹爪关节数（2） |
| `sim_substeps / frame_dt / sim_dt` | 时间参数 | 15 sub-steps, 60 fps |
| `cloth_solver_type` | `str` | `"vbd"` 或 `"style3d"` |
| `scene / multi_scene` | `ModelBuilder` | 场景构建 |
| `robot_solver` | `SolverMuJoCo` | 刚体求解器 |
| `cloth_solver` | `SolverVBD` 或 `SolverStyle3D` | 布料求解器 |
| `collision_pipeline` | `CollisionPipeline` | SDF 静水力学碰撞 |
| `_init_joint_q / _init_joint_qd` | numpy 数组 | 初始状态快照（reset 用） |
| `_init_particle_q / _init_particle_qd` | numpy 数组 | 初始粒子状态（reset 用） |
| `_single_arm_models` | `list[Model]` | 单臂模型（IK 用） |
| `shape_cfg / mesh_shape_cfg / urdf_shape_cfg` | `ShapeConfig` | 三种接触参数配置 |

**核心方法（按移植复杂度排序）：**

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| `_build_scene()` | **极高** | 整个场景构建流水线，约 130 行，几乎完整移植 |
| `load_asset()` | **极高** | 资产加载分发（URDF/mesh(rigid/cloth/primitive)），约 150 行 |
| `simulate()` | **高** | 双求解器步进循环（rebuild BVH → clear forces → collide → rigid step → cloth step → swap） |
| `reset()` | **高** | 全量/部分 reset，包含 joint_q/joint_qd/particle_q/particle_qd |
| `__init__(cfg)` | 中 | 调用 `_build_scene()`，初始化求解器、碰撞管线、control 数组 |
| `_pre_finalize_scene()` | 低 | 钩子，默认空实现 |
| `_post_build_scene()` | 低 | 钩子，默认空实现 |
| `_load_urdf_asset()` | — | **抽象方法**，由子类实现 |
| `_setup_viewer_camera()` | — | **抽象方法**，由子类实现 |

**`_build_scene` 流水线（直接移植）：**

```
1. 创建 ModelBuilder(self.scene)
2. 注册 MuJoCo 自定义属性
3. 如果 style3d，注册 Style3D 属性
4. 配置 default_shape_cfg（ke/kd/kf/mu）
5. 创建三种 ShapeConfig（通用/mesh/urdf）
6. 遍历 cfg.assets → load_asset()
7. multi_scene.replicate(scene, world_count=num_envs)
8. _pre_finalize_scene() 钩子
9. multi_scene.finalize() → self.model
10. 创建 state_0、state_1
11. FK 同步初始状态 → 复制到 control targets
12. 构建 CollisionPipeline
13. 构建 robot_solver (MuJoCo)
14. 构建 cloth_solver (VBD 或 Style3D)
15. 保存初始状态快照（_init_joint_q 等）
16. _post_build_scene() 钩子
17. capture() — CUDA Graph
```

**需要修改的地方（适配本项目）：**
- 配置来源从 `cfg.env.xxx`（Hydra）改为 `cfg.xxx`（dataclass）
- `load_mesh` 的导入从 `newton._src.geometry.utils` 改为检查是否有公共 API 替代
- 移除对 `transforms3d` 的依赖（当前未使用）
- 移除 `from ..utils.env_utils import` 中的 Hydra 相关函数

### 第三层：ClothEnvARX — ARX-X5 机器人定制

**文件：** `sim/env/cloth_env_arx.py`

**职责：** 实现 ARX-X5 特定的 URDF 加载、手指 SDF 构建、重力补偿。

**需要实现的抽象方法：**

1. **`_setup_viewer_camera()`** — 设置相机位置（考虑桌面高度）
2. **`_load_urdf_asset()`** — 完整的 ARX URDF 加载逻辑：
   - 加载 URDF 到独立 ModelBuilder
   - 识别手指 body（link6/link7/link8）
   - 对手指 mesh 构建 SDF，标记 HYDROELASTIC
   - 对非手指 shape 做凸包近似
   - 设置关节初始位置、target_pos、增益、力矩限制、armature
   - 可选 MuJoCo 重力补偿
   - finalize 单臂模型 → 保存到 `_single_arm_models[]`
   - 合并到主 scene

**可选的钩子覆写：**

3. **`_pre_finalize_scene()`** — 添加地面平面

**此层代码量约 120 行**，几乎直接移植，改动最小。

## 支撑模块

### 配置系统（`sim/config.py`）

用 Python dataclass 替代 Hydra/OmegaConf：

```python
from dataclasses import dataclass, field

@dataclass
class SimConfig:
    # 环境
    headless: bool = False
    use_graph: bool = True
    num_envs: int = 1
    num_robot: int = 2
    endeffector_id: int = 6
    num_arm_joints: int = 6
    num_gripper_joints: int = 2
    cloth_solver_type: str = "vbd"   # "vbd" 或 "style3d"
    table_height: float = 0.0
    gravity_compensation: bool = False
    max_viewer_worlds: int = 4
    assets: list = field(default_factory=list)

    # 组件
    controller: str = "joint_pd"     # "joint_pd" 或 "ik"
    renderer: str = "default"
```

**不使用 `eval()`，改用注册表：**

```python
CONTROLLER_REGISTRY = {
    "joint_pd": "sim.controller.JointPDController",
    "ik": "sim.controller.IKController",
}
RENDERER_REGISTRY = {
    "default": "sim.renderer.DefaultRenderer",
}
```

## 关键风险与决策

| 决策点 | 推荐 | 理由 |
|--------|------|------|
| `load_mesh` 的导入路径 | 使用 `newton._src.geometry.utils.load_mesh`（与源项目一致） | 公共 API 目前无替代，后续 Newton 升级可能暴露 |
| `num_envs` 支持 | 先只做 `num_envs=1` | `replicate()` + per-world offset 索引增加大量复杂度 |
| CUDA Graph | 先做，但加 `use_graph=False` 开关 | GPU 上预期可用，加开关方便调试 |
| `eval()` vs 注册表 | 使用 `importlib` 动态导入 | 比 `eval()` 安全，也保留了配置切换组件的灵活性 |
| 配置从哪来 | Python 代码硬编码初始 demo 配置 | 避免引入 YAML 解析依赖，后面可按需加 `from_yaml()` |
